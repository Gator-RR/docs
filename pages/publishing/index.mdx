# Publishing

To publish content to Verify it's important to understand the following concepts that are part of publishing to Verify.
 - Identity Registry
 - The Content Graph
 - Content Storage
 - Access Control and Licenses

## Identity Registry
 
This section covers how signing key pairs are registered to Verify, as well as how intermediate identities that act
on behalf of a root identity are created and managed.

## The Content Graph

This section covers the content graph smart contract, its methods, and metadata specification for registering signed
content in Verify.

## Content Storage

This section specifies how content should be stored to be compliant with the Verify protocol.

## Access Control and Licenses

This section covers how to create smart contract licenses that allow for access control on assets published on Verify.
